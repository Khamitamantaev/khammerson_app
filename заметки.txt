Khammerson App — монорепозиторий
Это мой монорепозиторий, где в одной папке лежат и фронтенд, и бэкенд. Всё управляется через pnpm.
Структура:
apps/web — фронтенд на React + Vite + TypeScript + Tailwind
apps/server — бэкенд на NestJS + TypeScript
Инфраструктура:
Всё завёрнуто в Docker с двумя независимыми окружениями:
Development: фронт на 3006, бэк на 4000, postgres на 5433, hot-reload через watch
Production: фронт на 3005, бэк на 4001, postgres на 5432
Текущий стек:
tRPC для связи между фронтом и бэком (полная типизация)
Zod для валидации
PostgreSQL + pg (пул соединений, прямые SQL запросы)
JWT аутентификация (в процессе)
Taskfile для автоматизации команд
На сервере:
Базовые tRPC роутеры (users, auth)
Контекст с пользователем
protectedProcedure для проверки авторизации
Глобальный модуль БД с DatabaseService
ConfigService для конфигурации
На клиенте:
tRPC клиент настроен
TrpcProvider с React Query
Тестовая страница для проверки
Что уже работает:
Оба окружения (dev/prod) запускаются без конфликтов
tRPC запросы проходят (users.getMany, users.create)
База данных подключается в обоих окружениях
Playground доступен в dev режиме
Планы:
1. Доделать аутентификацию на бэкенде (JWT + cookies)
2. Создать страницы логина/регистрации на фронтенде
3. Добавить защиту роутов
4. Новые модули и миграции БД
